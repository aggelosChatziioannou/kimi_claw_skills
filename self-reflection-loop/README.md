# Self-Reflection Loop

Continuous learning through post-task analysis.

## What It Does

After every completed task, Kimi-Claw automatically reflects on:
- What went well (success factors)
- What didn't work (issues encountered)
- Lessons learned (insights gained)
- How to improve (actionable changes)

## Why It Matters

- **Continuous Improvement:** Each task makes the next one better
- **Pattern Recognition:** Identify recurring issues
- **Documentation:** Lessons become institutional knowledge
- **Transparency:** Users see the learning process

## Quick Start

After completing any task:

```
🔄 **Self-Reflection:**

✓ **What Worked:**
- Factor that led to success

✗ **What Didn't:**
- Issue encountered

💡 **Lessons Learned:**
- Insight gained

🔮 **For Next Time:**
- Actionable improvement
```

## Reflection Types

### Short Form
For routine tasks:
```
🔄 **Quick Reflection:**
✓ Good: [success]
✗ Fix: [improvement needed]
💡 Next: [future action]
```

### Long Form
For complex projects:
```
🔄 **Post-Project Reflection:**
📊 **Metrics:** Time, success rate
✓ **Wins:** What succeeded
✗ **Challenges:** What was hard
💡 **Improvements:** How to get better
📝 **Action Items:** Updates to make
```

### Error Analysis
When things go wrong:
```
🔄 **Failure Analysis:**
❌ **What Happened:** Description
🔍 **Root Cause:** Why it failed
💊 **Immediate Fix:** Resolution
🛡️ **Prevention:** How to avoid
```
